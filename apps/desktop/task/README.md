# Handoff: Голосовой чат-бот (десктоп) на базе Claude Code CLI

## Как использовать этот пакет
Распакуйте zip в пустую папку проекта (или рядом с ним), откройте терминал в этой папке, запустите `claude` и вставьте промпт:

```
Прочитай design_handoff_voice_chat/README.md и design_handoff_voice_chat/ТЗ для Claude Code.md.
Реализуй это приложение полностью: Electron + Node.js + TypeScript.
Файл "Голосовой чат.dc.html" — HTML-прототип интерфейса (референс, не production-код):
воспроизведи его внешний вид и поведение точно, но кодом целевого стека.
Начни с каркаса проекта и машины состояний, потом UI, потом интеграции (Whisper, диаризация, claude CLI, Piper TTS).
После каждого крупного шага собери и проверь, что приложение запускается.
```

## Overview
Десктопное приложение-чатбот с голосовым вводом: пользователь нажимает кнопку микрофона и говорит; речь распознаётся локально (Whisper), делится по говорящим (диаризация), текст уходит в Claude Code CLI, ответ печатается в чат и озвучивается локальным TTS (Piper). UX — как голосовой режим ChatGPT, но чат остаётся видимым; голосовая панель — снизу.

## About the Design Files
`Голосовой чат.dc.html` — **дизайн-референс, созданный в HTML**: интерактивный прототип, показывающий внешний вид и поведение. Это НЕ production-код. Задача — воссоздать этот интерфейс в целевом стеке (Electron + React + TypeScript), используя его паттерны. Полное техническое ТЗ — в файле `ТЗ для Claude Code.md` (стек, пайплайн, структура проекта, требования).

## Fidelity
**High-fidelity.** Цвета, типографика, отступы, радиусы и состояния в прототипе — финальные; воспроизводить пиксель-в-пиксель.

## Interactions & Behavior
- Машина состояний: `idle → listening → thinking → speaking → idle` (детали и авто-пауза — в ТЗ).
- Клик по микрофону → listening: live-блок, транскрипт растёт пословно, метки спикеров в реальном времени.
- Стоп → сегменты фиксируются как отдельные сообщения (по одному на спикера) → thinking → ответ Claude добавляется в чат → speaking (озвучка, прерываемая стопом или новым нажатием микрофона).
- Enter в текстовом поле отправляет как «Спикер 1».
- Автоскролл ленты вниз при новых сообщениях.
- «+ Новый» очищает чат и создаёт разговор; клик по разговору в сайдбаре переключает его.
- Тумблер диаризации выключен → метки спикеров скрываются, live-подпись «Вы».

## State Management
- `voiceState`, `messages[{role: 'u1'|'u2'|'ai', text, time}]`, `liveSegments[{speakerId, text}]`, `conversations`, `activeConversationId`, `settings{model, whisperModel, diarization, voice, mic}`, `draft`.
- Persist: SQLite (разговоры/сообщения/настройки), session-id Claude CLI на разговор.
