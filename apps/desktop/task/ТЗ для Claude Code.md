# ТЗ: Десктопное голосовое чат-приложение на базе Claude Code CLI

## 1. Суть продукта
Десктопное приложение (Node.js + TypeScript), выглядящее как чат-бот. Пользователь нажимает кнопку и говорит; речь распознаётся **локально**, разделяется по говорящим (диаризация), текст отправляется в **Claude Code CLI** (`claude`), ответ озвучивается локальным TTS. UX — как голосовой режим ChatGPT, но чат остаётся видимым, голосовая панель снизу.

Референс UI: файл `Голосовой чат.dc.html` в этом проекте (интерактивный прототип).

## 2. Технологический стек
- **Оболочка:** Electron (main + renderer, contextIsolation, IPC через preload).
- **Язык:** TypeScript везде.
- **UI:** renderer — React (или по вкусу), вёрстка по прототипу.
- **STT (локально):** `whisper.cpp` через `smart-whisper` / `nodejs-whisper`, модель `large-v3-turbo` (настраиваемо: turbo / medium / small). Стриминговое распознавание по чанкам аудио.
- **Диаризация (локально):** `sherpa-onnx` (speaker diarization: сегментация + speaker embeddings, кластеризация). Альтернатива: python-подпроцесс с `pyannote.audio`, если качество sherpa не устроит. Результат — сегменты `{start, end, speakerId, text}` после совмещения с таймкодами Whisper.
- **LLM:** Claude Code CLI. Запуск: `claude -p "<текст>" --output-format stream-json --resume <session-id>` через `child_process.spawn`. Сессия CLI = один разговор в UI (хранить session-id на разговор).
- **TTS (локально):** Piper (`piper-tts`), русские голоса (irina/dmitri), стрим в аудиовыход по предложениям — начинать озвучку до конца генерации ответа.
- **Аудиозахват:** в renderer через `getUserMedia` + AudioWorklet (16 kHz mono PCM), передача чанков в main по IPC.
- **Хранение:** SQLite (better-sqlite3) — разговоры, сообщения, спикеры, настройки.

## 3. Пайплайн (машина состояний)
`idle → listening → transcribing → thinking → speaking → idle`

1. **listening:** кнопка микрофона нажата. Чанки PCM идут в Whisper (частичные гипотезы показываются в live-блоке «Распознавание») и в буфер диаризации. VAD (silero-vad через sherpa-onnx) для отсечения тишины.
2. **stop / авто-пауза:** пользователь нажимает стоп (или 2 с тишины в режиме hands-free). Финализация транскрипта + диаризация → сегменты по спикерам.
3. **thinking:** сегменты форматируются в промпт вида:
   ```
   [Спикер 1]: …
   [Спикер 2]: …
   ```
   и отправляются в `claude -p`. Стрим ответа парсится (stream-json) и печатается в чат по мере генерации.
4. **speaking:** готовые предложения ответа отправляются в Piper и проигрываются. Кнопка «стоп» прерывает озвучку. Новое нажатие микрофона прерывает speaking (barge-in).

## 4. UI (по прототипу)
- **Сайдбар:** список разговоров (заголовок + дата/кол-во сообщений), кнопка «+ Новый», внизу «Настройки».
- **Чат:** пузыри; сообщения пользователя справа с цветной меткой спикера (Спикер 1 — синий, Спикер 2 — зелёный, 3+ — новые цвета), ответы Claude слева с меткой «Claude». Время у каждого сообщения.
- **Live-блок при записи:** пунктирная карточка «Распознавание · локально (Whisper)» с растущим транскриптом по спикерам; строка «Обнаружено говорящих: …».
- **Голосовая панель (низ):** в idle — текстовый инпут + круглая кнопка микрофона; при записи — анимация волны + кнопка стоп; при thinking — индикатор «Запрос отправлен…»; при speaking — эквалайзер + «Claude отвечает голосом» + стоп.
- **Настройки (модалка):** модель Claude, модель Whisper, тумблер диаризации, голос TTS, выбор микрофона.

## 5. Требования
- Полностью офлайн-STT/TTS; в сеть выходит только Claude CLI.
- Задержка от конца речи до начала ответа Claude ≤ 2 с (turbo-модель, стриминг).
- Диаризация: до 4 спикеров, стабильные id спикеров внутри одного разговора.
- Текстовый ввод работает наравне с голосовым.
- Graceful-обработка: CLI не найден / не залогинен → понятная ошибка с подсказкой `claude login`; нет модели Whisper → предложить скачать.
- История сохраняется между запусками; удаление разговора.

## 6. Структура проекта (ориентир)
```
src/
  main/            # Electron main: ipc, claude-cli.ts, stt/, diarization/, tts/
  preload/
  renderer/        # React UI по прототипу
  shared/types.ts  # Message, Segment, ConversationState, Settings
models/            # whisper / sherpa / piper (гитигнор, скачивание при первом запуске)
```

## 7. Вне рамок v1
Hands-free непрерывный диалог без кнопки, имена спикеров по голосовым профилям, мультиязычное переключение на лету.
